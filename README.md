VERITAS is a security-focused compiler designed to detect vulnerabilities in C code during the compilation process itself. Unlike traditional compilers such as GCC or Clang that primarily focus on syntax correctness and performance optimization, VERITAS integrates security analysis directly into the semantic phase of compilation.
The core idea behind VERITAS is simple: code that looks correct is not always secure. Many critical vulnerabilities, such as buffer overflows, integer overflows, and use-after-free errors, often go unnoticed during standard compilation. VERITAS addresses this gap by analyzing the program’s structure and behavior before it reaches execution.
The system works by converting source code into an Abstract Syntax Tree (AST) and then performing pattern-based vulnerability detection on it. If any critical issue is identified, the compiler immediately halts the compilation process, preventing unsafe code from being generated. This ensures that potential security flaws are caught early in the development cycle rather than after deployment.
VERITAS also provides a side-by-side comparison between a traditional compiler and the secure compiler, helping users clearly understand where vulnerabilities exist and how they might be overlooked in standard workflows. This makes it not just a tool for detection, but also a learning aid for writing safer code.
The project includes a complete compiler pipeline — from lexical analysis and parsing to semantic analysis and code generation — along with a web-based interface powered by a REST API. Users can input C code, run it through the system, and instantly see whether it is secure or vulnerable.
VERITAS demonstrates how integrating security checks into the compilation process can significantly reduce the risk of software vulnerabilities and improve code reliability from the ground up.
